## 🤝 Contribution Guidelines

We welcome contributions to improve Lorexa. Please follow the guidelines below:

---

### 📌 Branching Strategy
- `main` → Stable production code  
- `develop` → Active development branch  
- `feature/<feature-name>` → New features  
- `bugfix/<issue-name>` → Bug fixes  

---

### 📝 Commit Message Format
All commits must follow a clear format:
<type>: <short description>

**Types:**
- `feat` – Add new feature  
- `fix` – Fix a bug  
- `ui` – UI/UX improvements  
- `docs` – Documentation updates  
- `refactor` – Code improvement  

**Example:**
feat: Add chat screen UI
fix: Resolve login validation issue


---

### 🔄 Pull Request Process
1. Create a new branch from `develop`  
2. Make changes and commit properly  
3. Push to GitHub  
4. Create a Pull Request  
5. Add description of changes  
6. Get approval before merging  

---

### 🧹 Code Guidelines
- Write clean and readable code  
- Follow consistent naming conventions  
- Avoid unused variables  
- Maintain proper structure and formatting  

---

### 🔒 Security Rules
- Do not commit API keys or passwords  
- Keep sensitive data secure  

---

### 🧪 Testing
- Test features before committing  
- Ensure app runs without crashes  
- Verify UI on different devices  

---
